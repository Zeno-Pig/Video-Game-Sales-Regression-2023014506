# 实验报告：基于 OLS 的电子游戏销量因素探索

## 1. 确定具体研究问题
在竞争激烈的游戏市场，发行商需要回答：**“在什么时间、什么平台、发布什么类型的游戏最可能获得成功？”**
(数据集来源:https://www.kaggle.com/datasets/gregorut/videogamesales/data)
本项目旨在量化：
1. 发行年份对销量的历史趋势影响。
2. 不同游戏类型之间的平均销量差异。
3. 不同硬件平台的市场溢价能力。

## 2. 选择与设计指标变量
### 2.1 因变量 (Y)
- **Log_Global_Sales**: 对 `Global_Sales` 进行 `ln1p` 对数变换。
  - *原因*：原始销量数据呈现极端右偏分布，对数变换能缓解异方差性，使残差更符合正态分布假设。

### 2.2 自变量 (X)
- **Year (年份)**：数值型变量，用于捕捉时间趋势。
- **Genre (类型)**：类别变量（进行独热编码）。
- **Platform (平台)**：类别变量（进行独热编码）。

## 3. 收集整理统计数据
- **样本量**：13,061 条有效记录。
- **数据清洗**：移除了年份缺失值 (N/A)，剔除了极少量非主流平台样本以保证统计稳定性。

## 4. 回归模型的数学形式
模型采用多元线性回归模型：
$$\ln(Global\_Sales + 1) = \beta_0 + \beta_1 Year + \sum \gamma_i Genre_i + \sum \delta_j Platform_j + \epsilon$$

## 5. 模型的求解：statsmodels 详细介绍
### 5.1 为什么选择 statsmodels？
在 Python 生态中，`statsmodels` 是专门为统计学设计的库。与 `scikit-learn`（侧重机器学习和预测精度）不同，`statsmodels` 的优势在于：
- **详尽的摘要表**：一次性提供 P值、标准误、t值、置信区间等所有统计指标。
- **诊断能力**：内置了检验多重共线性、自相关、异方差的专业统计检验工具。
- **统计学严谨性**：更适合进行假设检验（Hypothesis Testing）和推断分析。

### 5.2 参数估计结果
根据 OLS 求解：
- **Intercept (const)**: 39.8872 (P < 0.001)
- **Year Coef**: -0.0199。说明在其他因素不变的情况下，年份每增加一年，销量约下降 2%。
- **最优平台**: PS4 (0.6095) 和 GB (0.6681) 表现出最强的销量带动效应。

## 6. 模型的检验与修正
### 6.1 显著性检验
- **F-statistic**: 42.57 (Prob: 0.00)，表明模型整体在 1% 的显著性水平下成立。
- **多重共线性 (Condition Number)**: 1.81e+06，数值极高。
  - *修正说明*：这反映了年份与特定平台（如 PS4）之间的共生关系。在后续推广中，建议按代际（如 8位机、16位机、现代主机）对平台进行重新分组以降低共线性。

### 6.2 残差检验
- **Jarque-Bera (JB)**: 65759 (Prob: 0.00)，残差存在明显的尖峰厚尾特征，符合游戏市场“少数爆款统治销量”的特征。

## 7. 模型的应用与推广
### 7.1 应用方向
- **发行商决策**：在预算有限时，优先开发射击或平台类游戏，并选择安装量大的平台（如 PS4 对应的系数显著优于 PS3）。
- **市场基准**：该模型可作为新游戏的“预期销量基准”，实际表现超出模型预测的部分可归功于 IP 质量或营销。

### 7.2 推广局限
- 数据集主要反映实体销量，未包含现代数字分销（如 Steam, PSN 商店）的内购数据。
